Connect to rippled

server_info is a rippled command.

A response confirms that an application can reach rippled and indicates whether the rippled server is in sync with the rest of the network

rippled offers three ways to invoke commands such as server_info:

  • WebSocket
  • JSON-RPC
  • Command Line

Connect to rippled

Command line

rippled features are available through a command line interface, only when you are running an instance of rippled on your local machine.

To invoke the server_info command, for example: 


						rippled server_info

Connect to rippled

JSON-RPC

Applications may also POST Remote Procedure Calls (RPC) to rippled. 

curl -s -X POST "$RIPPLED_RPC" -d @- <<-END
{
    "method": "server_info",
    "params": [ {} ]
}
END

Connect to rippled

Websockets

WebSockets allow applications to submit multiple requests over a long-lived connection. Requests may receive multiple responses, or subscribe to events. To invoke server_info, POST: 

{
	"id": 1,
	"command": "server_info"
}

The response will include the same id that was passed in. Your application should give each request a unique id.

Connect to rippled

SDK's

Query Ledger Data

Account Info

The account_info command retrieves information about an account, its activity, and its XRP balance.

Most commands, including account_info, can be invoked via rippled WebSockets or JSON-RPC, or via JavaScript. 

curl -s -X POST "$RIPPLED_RPC" -d @- <<-END
{
    "method": "account_info",
    "params": [{ "account": "$ADDRESS" }]
}
END

Write Ledger Data

A transaction is the only way to modify the XRP Ledger.

Transactions are only valid if signed, submitted, and accepted into a validated ledger version following the consensus process

Write Ledger Data

There are several transaction types which have a variety of effects on the ledger.

Payment is the transaction type we'll show next...

Write Ledger Data

Payment Transaction

A Payment transaction represents a transfer of value from one account to another.

Payments are also the only way to create accounts.

The following slides will show how this is done...

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.on( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );

Take a Moment

Press B or . on your keyboard to pause the presentation. This is helpful when you're on stage and want to take distracting slides off the screen.
